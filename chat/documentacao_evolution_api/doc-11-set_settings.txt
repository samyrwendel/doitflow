## Introdução
Endpoint para definir as configurações de uma instância existente na Evolution API.

### Método
**POST**

### Endpoint
`/settings/set/{instance}`

### Exemplo de cURL
```bash
curl --request POST \
  --url https://{server-url}/settings/set/{instance} \
  --header 'Content-Type: application/json' \
  --header 'apikey: <api-key>' \
  --data '{
    "rejectCall": true,
    "msgCall": "<string>",
    "groupsIgnore": true,
    "alwaysOnline": true,
    "readMessages": true,
    "readStatus": true,
    "syncFullHistory": true
  }'
Respostas
Sucesso (201)
Código: 201

JSON

{
  "settings": {
    "instanceName": "teste-docs",
    "settings": {
      "reject_call": true,
      "groups_ignore": true,
      "always_online": true,
      "read_messages": true,
      "read_status": true,
      "sync_full_history": false
    }
  }
}
Erro (404)
Código: 404

Autorização
apikey (string) - requerido: Sua chave de autorização no cabeçalho.
Parâmetros de Caminho
instance (string) - requerido: Nome da instância para a qual as configurações estão sendo definidas.
Tipo de Body
application/json
Tipo de Resposta
201 - application/json

A resposta é do tipo object.
